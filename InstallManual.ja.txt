2. ファイル構成
------------------------------------------------------------------------

パッケージ(tmt-cups.tar.gz)の構成は、以下の通りです。

  InstallManual.ja.txt          インストールマニュアル。
                                本ファイルです。

  install.sh                    インストールスクリプトファイル。
                                プリンタードライバーをインストールするための
                                プログラムです。

  ppd                           PPDファイル格納ディレクトリ。
                                各機種毎のPPDファイルと、機種別のマニュアルが
                                格納されています。

  tmt-cups                      プリンタードライバーパッケージ格納ディレクトリ

  backend                       バックエンド及び通信モジュール格納ディレクトリ

3. 動作環境
------------------------------------------------------------------------

本ソフトウェアは、以下の環境をサポートしています。
サポート環境以外での動作については保証致しません。

* Operating System
  - openSUSE 11.1
  - Ubuntu 9.04

* Hardware:
  - Intel x86(32bit) architecture.
  - Intel x86_64(64bit) architecture.

* 関連ソフトウェア
  - CUPS Ver.1.1.23 以降
      <http://www.cups.org/>

    - Ubuntu 9.04の場合
      cupsパッケージは1.3.9-17ubuntu3.4以降が必要です。

  - libusb-1.0.0 以降
      <http://www.libusb.org/>

* 弊社では、以下の環境で動作確認を行っております。

  - openSUSE 11.1環境
     - libusb-1_0-0-1.0.2-2.2
     - cups-1.3.9-7.1

  - Ubuntu 9.04環境
    - libusb-1.0-0-2:1.0.0-1
    - cups-1.3.9-17ubuntu3.4

* ご利用の環境における上記パッケージの情報は
  以下の方法で確認できます。

  - openSUSE 11.1環境
    libusb-1_0-0の確認方法
      1.インストールされているパッケージ名の確認。
       $ rpm -qa | grep libusb-1_0-0
       libusb-1_0-0-1.0.2-2.2      ← 表示内容

      2.上記コマンドより、表示されたパッケージの詳細情報を確認。
       $ rpm -qi libusb-1_0-0-1.0.2-2.2

    cupsの確認方法
      1.インストールされているパッケージ名の確認。
       $ rpm -qa | grep cups
       ・・・
       cups-1.3.9-7.1              ← 表示内容
       ・・・・

      2.上記コマンドより、表示されたパッケージの詳細情報を確認。
       $ rpm -qi cups-1.3.9-7.1

  - Ubuntu 9.04環境
    libusb-1.0-0の確認方法
      $ dpkg-query -l libusb-1.0-0

    cupsの確認方法
      $ dpkg-query -l cups

4. インストール方法
------------------------------------------------------------------------
4.1 ご利用の環境に必要なパッケージをインストールして下さい。
  
  * 必要なパッケージを入手するために、ご利用の環境がインターネット
    に接続していることを確認してください。

    - openSUSE 11.1 環境の場合
      1. libusb-1_0-0のパッケージをダウンロードして下さい。

         openSUSE用のパッケージは、2010年3月現在では、以下のサイトから入手
         することが可能です。

         x86(32bit) architecture の場合
         <http://download.opensuse.org/distribution/11.2/repo/oss/suse/i586/libusb-1_0-0-1.0.2-2.2.i586.rpm>

         x86_64(64bit) architecture の場合
         <http://download.opensuse.org/distribution/11.2/repo/oss/suse/x86_64/libusb-1_0-0-1.0.2-2.2.x86_64.rpm>

      2. ダウンロードしたパッケージをインストールして下さい。

         例） x86(32bit)architecture の場合
           # rpm -i libusb-1_0-0-1.0.2-2.2.i586.rpm

         注）
           ! インストール作業には、root 権限が必要です。

    - Ubuntu 9.04 環境の場合
      1. 以下の方法でlibusb-1.0-0のインストールを実行して下さい。
         # apt-get install libusb-1.0-0=2:1.0.0-1


      2. CUPSパッケージのアップデートは以下の方法で実行して下さい。
         # apt-get install cups

      注）
        ! インストール作業には、root 権限が必要です。
        ! 上記の手順で必要パッケージがインストールされていない場合、
          下記のコマンドを実行後、再度 インストールを実行して下さい。
          # apt-get update

4.2 ドライバーのインストール

  * install.shを実行します。

    ファイルマネージャー等よりinstall.shのアイコンをダブルクリックするか、
    コマンドラインプロンプトより以下のように実行して下さい。

    # ./install.sh

  注）
    ! インストール作業には、root 権限が必要です。
      root以外で実行した場合は、install.shによりsudoコマンドが
      実行されますので、パスワードを入力して下さい。
    ! ドライバーをインストールする前に、プリンターの電源をOFFにして下さい。

  * インストールするディストリビューションの情報が端末上に表示されますので、
    間違いなければ"y"を入力後、Enterキーを押下して下さい。

    例) openSUSE 11.1 (i586)の場合
    # Install openSUSE-11.1 (i586)  [y/n]?      ← メッセージ

  * 表示されたディストリビューションが、インストールされているものと異なる
    場合は、"n"を入力後、Enterキーを押すとディストリビューションの選択画面が
    表示されます。1から4の数字を入力後、Enterキーを押下して下さい。

    Please select your distribution.
    1.Ubuntu 9.04 i586 DEB
    2.Ubuntu 9.04 x86_64 DEB
    3.openSUSE 11.1 i586 RPM
    4.openSUSE 11.1 x86_64 RPM
    Select number [1/2/3/4]?

  * インストールが実行されます。

5. 設定方法
------------------------------------------------------------------------

  * デバイスポートの設定

    USB以外のポートを使用する場合は、ポートの設定が必要です。
    /opt/epson/epuras/EPPortConfig を実行してください。

    詳細はバックエンドのマニュアルを参照して下さい。
    バックエンドのマニュアルは、本パッケージのbackendディレクトリ内に
    格納されています。
    
    注）
    ! ポートの設定はプリンターの電源をOFFにした状態で実施して下さい。
    
  * CUPS へのプリンター登録

    CUPSへプリンターを登録する前に、プリンターの電源をONにして下さい。

    WebブラウザーからCUPS WebAdmin <http://localhost:631/>を開き、
    プリンターをCUPSに登録して下さい。

    CUPS の詳細は、下記 URL から参照できます。
          <http://localhost:631/help/>
          <http://www.cups.org/>

    - プリンターのデバイス選択画面で、"Epson TM/BA/EU Printer"を選択して下さい

      - USB TM-T88IVの例
        Epson TM/BA/EU Printer (USB TM-T88IV)

    - プリンターのモデル選択画面では、直接PPDファイルを選択して設定を行って
      下さい。
      PPDファイルは、本パッケージのppdディレクトリ内に格納されています。

    注）
    ! ディストリビューション付属のプリンタ設定ツールにより自動認識画面が
      表示された場合には、キャンセルして下さい。

  * 印刷設定

    プリンタードライバーのご利用方法については、プリンタードライバー説明書を参照下さい。
    プリンタードライバー説明書は、ディストリビューションによりインストールされる
    場所が若干異なりますので、ご注意下さい。

    例) openSUSE 11.1の場合
    /usr/share/doc/packages/tmt-cups/README.ja.txt (日本語)
    /usr/share/doc/packages/tmt-cups/README.en.txt (英語)

    使用できる印刷設定は、プリンターのPPDに付属のマニュアルを
    参照して下さい。

    例) TM-T88IVの場合
    TM-T88IV_Manual.ja.txt (日本語)
    TM-T88IV_Manual.en.txt (英語)

7. 注意／制限事項
------------------------------------------------------------------------

* 古いドライバーをアンインストールせずにバージョンアップを行うとドライバーが
  動作しなくなります。

  バージョンアップの前に古いドライバーをアンインストールしてください。


* rpm -e でアンインストールする際に、パッケージを一括して指定すると、
 失敗することが有ります。

  以下のパッケージの順にアンインストールを実行してください。
  tmt-cups
  epson-cups-escpos
  ep-escpos
  ep-core
  ep-client

  また、rpmやdpkgコマンドでエラーが発生した場合には、エラーメッセージに
  従って操作をやり直してください。


* ジョブキャンセルをしても、印刷はすぐに止まりません。

  いったんプリンターから印刷しはじめたジョブは、すぐにキャンセルできません。
  ジョブキャンセル後、プリンターの印刷が終了するまではプリンターの電源を
  切ったり、プリンターケーブルを外さないで下さい。


* プリンターの電源を入れるとごみ印字が発生することがあります。

  印刷中はプリンターの電源を切ったり、通信ケーブルを外さないで下さい。
  また、通信エラー等で印刷が中断された場合には、印刷していたジョブを
  削除してから、プリンターの電源を再投入して下さい。


* 印刷中および印刷待ち状態で、サスペンド、ハイバーネートしてしまった場合には、
  印刷が再開されないことがあります。

  サスペンド、ハイバーネートが発生しないようにして下さい。


* ディストリビューション固有のプリンター管理ツールからプリンターを登録した
  場合、印刷に失敗する場合があります。

  プリンター登録は、CUPS WebAdmin から行って下さい。


* Internal Buzzer選択時、Buzzer Repeatは2以上を選択しても繰り返し鳴りません。

  Internal Buzzer選択時、Buzzer Repeatは1のみが有効です。


* プリンターを停止させて溜めた印刷ジョブを一度に開放すると、
  印刷が停止する場合があります。

  そのような操作の後で印刷が停止した場合は、印刷ジョブを削除して
  PCを再起動してください。


* 印刷結果に横スジが現れる場合があります。


* シリアルで接続されたプリンターで印刷中に、用紙交換などのためにカバーを開け
  閉めするとゴミが印刷されます。

  その場合には印刷していたジョブを削除してから、プリンターの電源を再投入して
  下さい。


* プリンターをハードウェアフローコントロールのシリアルで接続している場合、
  正常に印刷できない場合があります。

  プリンターをシリアルで接続する場合はXon/Xoffフロー制御を使用してください。


* プリンターの置き換え後に正常に印刷できない場合があります。

  プリンターの置き換え後にPCを再起動してください。


* Ethernetで接続されたプリンターの電源を入れる前に実行した印刷は、
  プリンターの電源を入れた後印刷されません。

  Ethernetで接続している場合、プリンターの電源を入れた後に印刷するよう
  にしてください。
